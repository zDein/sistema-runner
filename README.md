# 🖋️ Projeto Sistema Runner

Este repositório é dedicado ao desenvolvimento de um componente de assinatura digital, integrando rigor técnico em modelagem de software com as melhores práticas de implementação.

---

## Guia de Navegação do Projeto

Abaixo estão os pilares que sustentam a evolução e a documentação deste sistema:

### [Diário de Bordo](./docs/diario-de-bordo.md)
> **Registro de Evolução e Insights**
> Local onde documento a trilha de execução. Mais do que um log de tarefas, aqui registro o raciocínio aplicado na produção de artefatos, os desafios técnicos superados e as lições aprendidas durante o processo.

### [Backlog do Produto](./docs/backlog.md)
> **Gestão de Entregáveis e Prioridades**
> Definição clara do que deve ser construído. Contém a descrição técnica, critérios de aceitação e a ordem de prioridade de cada funcionalidade e requisito do sistema.

### [Modelos e Arquitetura](./docs/modelos/README.md)
> **Domínio do Problema e da Solução**
> Onde a abstração encontra a estrutura. Este espaço armazena:
> * **Modelagem de Domínio:** Diagramas e artefatos que auxiliam na compreensão do problema.
> * **Design de Solução:** Decisões de arquitetura (ADRs), diagramas de componentes e padrões (ex: Hexagonal) que definem como o software resolve o problema.

## [Estudos](./estudos/)
> **Diretório com projetos em Java feitos para aprendizado da linguagem**
> Local onde armazeno tudo o que estou aprendendo sobre a linguagem para ter um domínio mínimo da linguagem Java.

### [Sistema Runner](./src/runner/README.md)
> **Núcleo de Execução**
> Onde o código ganha vida. Este diretório contém a implementação técnica do componente assinador, estruturado para ser modular, testável e eficiente.
